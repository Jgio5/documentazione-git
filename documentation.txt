Git - Controllo di versione
Sistema che registra nel tempo uno o più file modificati da consultare nel tempo

Ogni client è una copia di backup dell'archivio,
si può puschare in un secondo momento lavorando in locale.

git --version (verificare se è installato)

git config --list (vedere utenti e impsotazioni salvate)

git help config | git config --help (tutti i comandi)

git init (creare cartella git nascosta)

git clone (url git) nome-cartella (Scarico da github un progetto e lo clono creando una nuova cartella)

____________

git status (stato attuale dei file)
git status -u (fa vedere i file dentro la directory)

git add (contrassegnare un file nella nostra area)
git restore --staged nome-file (posso toglierlo da add)

git commit (caricare una istantanea)
Inserire in alto dove non c'è # il commento del commit
Es.: Essendo il primo inserico: Initial commit
OPPURE
git commit -m "Initial commit"

git log (registro delle istantanee)
git log --stat (indica quante riga sono state aggiunte o cancellate)
git log --oneline (tutti i commit, 1 per riga)

_____________

git commit --amend (sovvrascrive il commit precedente e cambiare la nota)
git commit --amend -m "add license"

git restore --staged nome-file (posso toglierlo da add)
git restore nome-file (riportiamo il file all'ultimo salvataggio)

git diff (file modificati rispetto all'ultimo commit e cosa è stato modificato)
git diff nome-file (per vedere modifiche di solo quel file)

____________

git log --oneline --all --graph (mi fa un'alberatura dei commit)

____________

git remote (Per collegarci in remoto, se è vuoto vuol dire che non c'è nulla)

git remote add origin (alias) url-repository
git remote
origin

git remote --verbose | git remote --v (url della repo)

git push origin (alis) master (nome ramo) (carico i file su github)

git remote show origin (dettagli repo)

git fetch --all (vado a scaricare gli aggiornamenti dalla repo remota)

git log --oneline --all (per vedere anche lo storico della repo remota)

git pull origin master (sincronizzazione del ramo master con il ramo remoto)

git remote rename backup bck (rinomina l'alias remoto)
git remote rm bck (Elinare repo remoto)

_____________

git branch carbonara (crea un nuovo ramo)
git checkout carbonara (mi sposto su un altro ramo in base al nome che scelgo)

git checkout master (i merge vanno sul principale)
git merge risotto (per unire i due rami)
git branch -d risotto (cancello il puntatore)
_____________

1. Aggiungo un file con un altro utente
git rebase master (Attenzione a non farlo da remoto se non sei sicuro che non siano allineati i profetti con altri sviluppatori)
2. In questo caso unifico le modifiche degli utenti in un solo ramo

Come ignorare alcuni file o directory?

# commenti
! negazioni
/ directory

Se faccio:

idee/ (escludo tutta la cartella per cui non potrò fare eccezzioni)
idee/* (escludo tutti i file dentro la cartella ma non la cartella, in questo modo creo le eccezioni)
!idee/istruzioni.md (escludo)

_____________

Sospendere e riprendere una modifica più avanti
git stash push (push facoltativo)
git stash list (per vedere l'indice dello stash)
git stash apply stash@{0} (per tornare allo stash)
git stash drop stash@{0} (serve per eliminare lo stash, se sono di più git stash clear)

_____________

git tag -a v1.0 dcdaf45 -m "Release 1.0" (Creo un tag in maniera locale)
git tag v1.0 8d61829 (se non si vuole scrivere un commento)
git show v1.0 (Per visualizzare il tag)
git push origin v1.0 (per pushiare il tag)

git tag v1.1 (crea il tag sull'ultimo commit)
git tag -d v1.1 (eliminare il tag)
